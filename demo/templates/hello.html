How it Works
  1. User logs in via the demo app
  2. Demo app calls Auth Service login API
  3. Auth Service returns JWT token
  4. Demo app stores token in session
  5. All subsequent requests include the token
  6. Auth Service validates token on each request
Architecture
  • Demo App: This application (Port 5001)
  • Auth Service: Standalone auth service (Port 5000)
  • Database: Shared PostgreSQL database
  • Redis: Session management and caching
  • Nginx: Reverse proxy and load balancing
